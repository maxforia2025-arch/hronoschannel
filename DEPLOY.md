**Осталось ровно два действия Президента: BOT_TOKEN и публичный адрес канала.**

> ⚠️ **Публичный адрес.** Канал пока приватный, публичный адрес не задан.
> Постинг работает по числовому id — но в тексте каждого поста стоит `@hronos_daily`,
> и пока адрес не занят, это **нерабочая ссылка у подписчиков**. Займи адрес в настройках
> канала (или впиши в `config.json` → `channel_handle` тот, что реально занял) **до**
> первого поста. Это же значение идёт в секрет `CHANNEL_ID`.

## Сделать один раз (~10 минут)

1. **Адрес канала.** Настройки канала → тип «Публичный» → занять адрес (проект: `hronos_daily`).
   Если занят другой — вписать его в `config.json` → `channel_handle`.
   Заодно поставить аватар `../brand/logo_avatar.png`, описание и закреп — всё готовыми
   блоками лежит в `../brand/BRAND.md`.
2. **Бот-постер.** У **@BotFather** → `/newbot` → получи **BOT_TOKEN**.
   Добавь бота **администратором** канала с правом «Публиковать сообщения».
3. **Репозиторий.** Создай приватный репо `hronos-bot` и залей туда содержимое
   папки `engine/` (именно её содержимое — `.github/` должен оказаться в корне репо):
   ```bash
   cd "hronos_channel/engine" && git init && git add . && git commit -m "hronos engine"
   git branch -M main
   git remote add origin https://github.com/<твой-аккаунт>/hronos-bot.git
   git push -u origin main
   ```
4. **Секреты.** В репо → **Settings → Secrets and variables → Actions → New repository secret**:
   - `BOT_TOKEN` — токен из шага 2
   - `CHANNEL_ID` — `@hronos_daily` (твой хендл) **или** числовой `-100…`
   - `ANTHROPIC_API_KEY` — ключ Anthropic (для автопополнения банка)
   - `FUNNEL_BOT_USERNAME` — *(необязательно)* username общего `funnel_bot` без `@`,
     чтобы в постах появлялась ссылка на лид-магнит с меткой `src=hrn`
5. **Проверка.** Вкладка **Actions → hronos-post → Run workflow** (`count=1`).
   В канал должен упасть один пост. Дальше — по расписанию само.

## Наполнение банка

**Банк уже наполнен: 35 постов**, все прошли редстандарт канала (только документированные
события, точные даты, спорное проговаривается как спорное). При трёх постах в день это
почти 12 дней автономной работы — запас на прогрев с большим зазором.

Дальше банк поддерживает себя сам: воркфлоу `hronos-refill` держит ≥ 20 непоказанных
постов (порог `bank_min` в `config.json`), для этого ему нужен секрет `ANTHROPIC_API_KEY`.
Без ключа канал не встанет — просто перестанет пополняться, когда 35 постов кончатся.

Добавить эпизоды руками: заполнить в `../launch/CONTENT_QUEUE.md` и запустить
`python3 fill_bank.py` — очередь превратится в посты.

## Локальные команды

```bash
python3 post.py               # dry-run: показать следующий пост, ничего не слать
python3 post.py --status      # что в банке
python3 post.py --send        # отправить (нужны BOT_TOKEN/CHANNEL_ID в .env или ENV)
```

> Секреты в коде отсутствуют. Локально — в файле `.env` рядом со скриптом
> (он в `.gitignore`), в облаке — только GitHub Secrets.
